UI Design Overview

Given that our project is targeted to a wide range of users, from programmers to designers — all with different levels of experience — we designed a UI that caters to these groups. A careful balance was considered to create a UI that allows experienced designers ample avenues for customization while keeping it intuitive for general users.

We referenced **popular design products** such as Wix and Canva, using **Jakob Nielsen’s 10 Usability Heuristics** to guide our design process.

Design Principles

We followed several key usability principles to ensure an optimal user experience:

Design Feature Usability Heuristic Implementation in BlueprintAI
Customization Sidebar Recognition rather than recall All fields and labels are clearly visible and well-labeled for easy access.
Flexibility and efficiency of use Users can upload company logos, drawings, and other assets for customization.
Aesthetic and minimalist design Only relevant information is displayed in the sidebar to maintain clarity.

Additionally, we took inspiration from Wix and Canva’s **top bar design**, which contains crucial project-level controls.

Design Feature Usability Heuristic Implementation in BlueprintAI
Project-Level Top Bar Visibility of system status Flags notify users when they are in editing mode and when work is saved.
Match between system and real world Familiar icons such as undo, redo, and download buttons ensure usability.
User-controlled freedom Clearly marked undo/redo and exit buttons provide intuitive navigation.

AI Integration in UI

Unlike traditional UI builders, **BlueprintAI deeply integrates AI** into the interface design process. Instead of relegating AI to the corner (as seen in Wix/Canva), our **AI Chatbox System** is **prominently featured** while maintaining easy-to-use customization features.

Design Feature Usability Heuristic Implementation in BlueprintAI
AI Chatbox Error prevention Logs prompts and tracks UI changes, allowing for rollbacks.
Help users recognize and recover from errors AI highlights overflowing elements and potential UI conflicts.
Help and documentation Chatbox allows easy search and retrieval of features via AI queries.

Sketches

We created several hand-drawn **sketches** in the early stages of the project, allowing for rapid iteration based on user feedback.

Design A

**AI prompter positioned on the side** with a **portrait chat log**. This design was inspired by chatbot-style UIs such as ChatGPT.

Design B

**AI prompter positioned at the bottom**, and chat history accessed through a button.

Final Decision

Users preferred **Design A**, as it provided clearer AI responses while showing UI changes in real time. Improvements included:

  • Adding a **confirmation button** for AI-generated changes.
  • Keeping **all essential buttons** (download, pages, widgets) on one side for **better organization**.
  • Adding a **search bar** for improved usability.

Evaluation of Prototype

Severity Ratings:

  • 0: Not a usability problem
  • 1: Cosmetic problem (low priority)
  • 2: Minor usability issue (fix if time permits)
  • 3: Major usability issue (high priority)
  • 4: Critical usability issue (must be fixed before launch)
